## Spacing Scale

Tailwind v4 uses `--spacing: 0.25rem` (4px base). All values are multiples of 4px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (spacing-1) | Icon gaps, inline padding within badges |
| sm | 8px (spacing-2) | Compact element spacing, gap between form label and input |
| md | 16px (spacing-4) | Default element spacing, `space-y-4` within form sections |
| lg | 24px (spacing-6) | Section padding (`p-6` in CardContent, CardHeader) |
| xl | 32px (spacing-8) | Page-level vertical padding (`py-8` in Layout main) |
| 2xl | 48px — | Major section breaks between form Section 1 and Section 2 (use `space-y-12` = 48px) |
| 3xl | 64px (spacing-16) | Top/bottom padding on auth-style pages (`py-16` in LoginPage pattern) |

Exceptions:
- Nav bar height: 56px (`h-14`) — established in Layout.tsx; do not change
- Form max-width: `max-w-2xl` (672px) for the submission form (wider than auth pages' `max-w-md` to accommodate the two-section layout with multiple fields)
- Touch targets (Checkbox, RadioGroup items): minimum 44px height via `py-3` padding on each item row

## Color

Source: dist CSS color tokens + Layout.tsx + existing page classes. No custom CSS variables exist — all colors are Tailwind v4 zinc/white semantic tokens.

| Role | Tailwind Class | Hex Equivalent | Usage |
|------|---------------|----------------|-------|
| Dominant (60%) | `bg-zinc-50` / `bg-white` | #fafafa / #ffffff | Page background (`bg-zinc-50`), header and card surfaces (`bg-white`) |
| Secondary (30%) | `bg-zinc-100` / `border-zinc-200` | #f4f4f5 / #e4e4e7 | Input backgrounds on focus, dividers, disabled states |
| Accent (10%) | `text-blue-600` / `bg-blue-600` | oklch(54.6% .245 262.881) | Primary CTA button (submit), nav links (active state hover), hyperlinks only |
| Destructive | `text-red-600` / `bg-red-100` | oklch(57.7% .245 27.325) | Inline field error messages only |

Accent reserved for:
1. Primary submit button (`Button` default variant = `bg-primary` which resolves to zinc-900 in the zinc preset — do not override to blue; see note below)
2. Text links within helper copy (`text-blue-600 underline` — per LoginPage/RegisterPage pattern)
3. Focus ring on combobox and date inputs

**Important note on Button color**: The shadcn default variant uses `bg-primary` which in the zinc preset resolves to `bg-zinc-900` (near-black). This is the established CTA button color across all existing pages. Do NOT change the primary button to blue — maintain `bg-zinc-900 text-white` as the submit button appearance.

Status badge colors (established in Phase 3, do not change):
- Fulfilled (Կատարված): `bg-green-100 text-green-800 border-green-200`
- Broken (Խախտված): `bg-red-100 text-red-800 border-red-200`
- In Progress (Ընթացքի մեջ): `bg-yellow-100 text-yellow-800 border-yellow-200`
- Stalled (Կասեցված): `bg-yellow-100 text-yellow-800 border-yellow-200`
- Not Rated (Չգնահատված): `bg-zinc-100 text-zinc-600 border-zinc-200`

## Component Inventory

### PromiseSubmitPage (`/promises/new`)

**Layout**: `max-w-2xl mx-auto px-4 py-8` — wider than auth pages to accommodate the full form.

**Page structure**:
```
<main className="max-w-2xl mx-auto px-4 py-8">
  <Card>
    <CardHeader>
      <CardTitle>Ներկայացնել խոստում</CardTitle>
    </CardHeader>
    <CardContent>
      {/* Section 1: Ով */}
      <section aria-labelledby="section-who">
        <h2 id="section-who" className="text-lg font-semibold mb-4">Ով</h2>
        {/* Politician combobox (required) */}
        {/* Election toggle (optional) */}
        {/* Election cascading picker (conditional) */}
      </section>

      <Separator className="my-8" />

      {/* Section 2: Ինչ + Երբ */}
      <section aria-labelledby="section-what">
        <h2 id="section-what" className="text-lg font-semibold mb-4">Ինչ + Երբ</h2>
        {/* Title, quote, source URL, made_date, expected_date, description */}
      </section>

      {/* Submit or eligibility gate */}
    </CardContent>
  </Card>
</main>
```

**Politician combobox**:
- Uses shadcn Combobox pattern: `Popover` + `PopoverTrigger` (Button, outline variant, full-width) + `PopoverContent` + `Command` + `CommandInput` + `CommandList` + `CommandItem`
- Trigger placeholder: "Ընտրել քաղաքական գործչին"
- Empty search state in CommandList: "Ոչ մի արդյունք"
- Selected state: shows politician name_hy in the trigger button

**Election toggle**:
- `Checkbox` + `Label` in a flex row: `<Checkbox id="link-elections" />` + `<Label htmlFor="link-elections">Կապել ընտրություններին</Label>`
- When unchecked: election picker is hidden (`hidden` class, not unmounted — preserves scroll position)
- When checked: cascading picker appears below with `mt-4` spacing

**Election cascading picker**:
- Level selector: `RadioGroup` with 3 `RadioGroupItem` + `Label` pairs, horizontal layout (`flex gap-6`)
  - "Ազգային" / "Տեղական" / "Հանրաքվե"
- Election list (after level selected): vertical list of `Checkbox` + `Label` pairs, `space-y-2`, filtered client-side
- No level selected → election list shows instruction text: "Ընտրեք մակարդակ" (14px, zinc-500)

**Form fields (Section 2)**:
All fields follow Label → Input stacked pattern with `space-y-2` (8px) between label and input, `space-y-4` between field groups.

| Field | Component | Label | Required | Validation |
|-------|-----------|-------|----------|------------|
| Վերնագիր (Title) | `Input` type=text | Վերնագիր * | Yes | min 5 chars; required error: "Վերնագիրը պարտադիր է" |
| Մեջբերում (Verbatim quote) | `<textarea>` with `Input`-equivalent styling | Բառացի մեջբերում * | Yes | min 10 chars; required error: "Մեջբերումը պարտադիր է" |
| Աղբյուրի URL (Source URL) | `Input` type=url | Աղբյուրի հղում * | Yes | must start with http/https; error: "Անվավեր հղում" |
| Խոստման ամսաթիվ (Made date) | `Input` type=date | Խոստման ամսաթիվ | No | date format validation |
| Կատարման ամսաթիվ (Expected date) | `Input` type=date | Սպասվող կատարման ամսաթիվ | No | must be after made_date if both set |
| Նկարագրություն (Description) | `<textarea>` with `Input`-equivalent styling | Նկարագրություն | No | — |

**Textarea styling**: `className="flex min-h-[80px] w-full rounded-md border border-input bg-background px-3 py-2 text-sm ring-offset-background placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50 resize-y font-normal"` — mirrors shadcn Input styles with `resize-y` added; uses `font-normal` (400) weight.

**Inline field errors**: `<p className="text-sm text-red-600 mt-1">{errorMessage}</p>` — directly below the affected field. Source: LoginPage.tsx pattern.

**Submit area**:
- Eligible user: `<Button type="submit" className="w-full mt-6" disabled={isSubmitting}>{isSubmitting ? "..." : "Ներկայացնել"}</Button>`
- Ineligible user (verified=false OR account_age < 7): submit button is REPLACED (not disabled) with:
  ```
  <p className="text-sm text-zinc-500 mt-6 p-4 bg-zinc-100 rounded-md">
    Ձեր հաշիվը պետք է ունենա նվազագույնը 7 օրվա վաղեմություն և
    հաստատված էլ. հասցե՝ խոստում ներկայացնելու համար
  </p>
  ```
  Source: CONTEXT.md D-02 (exact copy).

## Accessibility Requirements

- All form fields must have a corresponding `<Label>` with `htmlFor` matching the field `id` — enforced via shadcn Label component.
- Combobox must be keyboard navigable: Enter to open, arrow keys to navigate, Enter to select, Escape to close.
- Election RadioGroup: keyboard navigable with arrow keys between options.
- Election checkboxes: Space to toggle, Tab to move between items.
- Error messages must use `aria-live="polite"` or be associated with the field via `aria-describedby`.
- Section headings (`<h2>`) use `id` attributes; sections use `aria-labelledby` pointing to those ids.
- The eligibility gate message must be in the DOM (not `display:none`) so screen readers announce it when rendered.
