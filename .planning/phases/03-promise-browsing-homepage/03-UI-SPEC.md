## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline icon-text gap |
| sm | 8px | Compact element spacing, filter row item gap |
| md | 16px | Default element spacing, card body padding, page horizontal padding |
| lg | 24px | Section padding, gap between filter row and results list |
| xl | 32px | Gap between page header and content area |
| 2xl | 48px | Major section breaks (e.g., quote hero from metadata row below it) |
| 3xl | 64px | Not used in Phase 3 |

Exceptions:
- Touch targets for pagination prev/next buttons: minimum 44px height (carried forward from Phase 2)
- Nav header fixed height: 56px (carried forward from Phase 2)
- Quote hero on promise detail: minimum 48px top + bottom padding to give it visual weight
- Stats block on homepage: 40px min-height per stat card

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Promise list item quote text, metadata labels, about page body text |
| Label | 14px | 600 (semibold) | 1.4 | Badge text, filter labels, nav links, stat labels |

**Quote hero special rule:** `quote_hy` on the promise detail page uses Display size (28px) at weight 400 (regular, not semibold) with line-height 1.5 to preserve readability for long verbatim text. This is the only Display-role text with regular weight.

## Color

Base color palette: **zinc** (from `components.json` `tailwind.baseColor: "zinc"`). shadcn/ui CSS variables active. All color values correspond to zinc-scale Tailwind tokens.

| Role | Tailwind Token | Hex (approx) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | zinc-50 / white | #fafafa / #fff | Page background, main content area |
| Secondary (30%) | zinc-100 / white | #f4f4f5 / #fff | Nav header, cards, filter row, stats block background, about page section bg |
| Text primary | zinc-900 | #18181b | Headings, promise title, politician names, stat numbers |
| Text muted | zinc-500 | #71717a | Meta info (politician name on promise list, made_date, election name) |
| Border | zinc-200 | #e4e4e7 | Card borders, dividers, promise list row borders |
| Accent (10%) | zinc-900 | #18181b | Logo text, active nav link, focus ring on interactive elements |

Accent reserved for: **logo text**, **active/hover nav link**, **focus ring on interactive elements** (keyboard navigation). Accent is NOT used for backgrounds, badges, decorative elements, or the "Աղբյուր" source link.

## Interaction Contracts

### Filter Behavior on `/promises`

- Status Select: writes `?status=<value>` to URL; resets `?page=1`
- Politician Select: writes `?politician_id=<uuid>` to URL; resets `?page=1`
- Election Select: writes `?election_id=<uuid>` to URL; resets `?page=1`
- Date inputs: onChange writes `?made_date_from=`, `?made_date_to=`, `?expected_date_from=`, `?expected_date_to=` ISO date strings; resets `?page=1`
- All filters via `useSearchParams` + `setSearchParams` — no React `useState` for filter values (carry-forward from Phase 2 D-04)
- "Reset filters" not required in Phase 3 MVP; each select has an "all" default option to individually clear

### Promise List Row Click

- Entire PromiseCard row is clickable: `<Link to={/promises/${slug}}>`
- Hover: `bg-zinc-50` on the row
- Cursor: pointer
- Status badge is not a separate clickable element

### Source Link Behavior

- "Աղբյուր" button: `<a href={source_url} target="_blank" rel="noopener noreferrer">`; shadcn Button variant="outline"
- "Wayback Machine" button: same pattern with `archived_url`; shadcn Button variant="ghost"
- Both open in new tab; rel="noopener noreferrer" required (security)

### Pagination — unchanged from Phase 2

- State stored in URL `?page=N` via `useSearchParams`
- Pagination component centered: `flex justify-center mt-6`
- Prev disabled (aria-disabled) at page=1; Next disabled at totalPages

### Loading States

- All data-fetching pages show skeleton placeholders while `isLoading === true`
- Skeleton: `animate-pulse` bg-zinc-200 rounded blocks matching content shape
- HomePage: stats block skeleton (5 boxes) + 10 promise stub skeleton rows
- PromisesListPage: filter row skeleton (3 selects + 4 date inputs) + 20 promise card skeleton rows
- FulfilledPage / UnfulfilledPage: 20 skeleton rows
- PromiseDetailPage: quote hero skeleton (large block) + 4 metadata row skeletons

### Error States — consistent with Phase 2

- When `isError === true`: inline error message (not full-page error)
- Error message: zinc-50 bg, red-600 text, `AlertCircle` lucide icon, 14px
- Error copy: see Copywriting section

### OG Tags (DISC-05)

OG tags are served by FastAPI for bot user-agents (D-15). This is a backend + proxy concern, not a frontend React interaction pattern. The React SPA requires no OG-related code. The bot-detection routing in Docker Compose / Nginx routes bot user-agents to `GET /api/og/promises/:slug` which returns a minimal HTML page. Regular browsers continue to the SPA.
