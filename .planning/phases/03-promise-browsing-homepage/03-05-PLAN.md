    nginx.conf content: use `map` directive for bot detection + `if ($is_bot)` inside location block. Exact structure:

    Top-level map block (outside server block):
    ```
    map $http_user_agent $is_bot {
        default         0;
        "~*Facebookbot" 1;
        "~*Twitterbot"  1;
        "~*TelegramBot" 1;
        "~*LinkedInBot" 1;
        "~*Googlebot"   1;
    }
    ```

    Server block:
    ```
    server {
        listen 80;

        location ~* ^/promises/(.+)$ {
            if ($is_bot) {
                proxy_pass http://backend:8000/api/og/promises/$1;
                break;
            }
            proxy_pass http://frontend:5173;
        }

        location / {
            proxy_pass http://frontend:5173;
        }
    }
    ```

    Key points for the location blocks:
    - `~*` for case-insensitive regex matching
    - `$1` captures the slug from the URL path
    - Non-bot /promises/{slug} requests fall through to the SPA at frontend:5173
    - All other routes (/, /fulfilled, /about, etc.) go directly to frontend:5173

    Update docker-compose.yml: add nginx service block AFTER the frontend service block (before the `volumes:` top-level key). Service definition:
    ```yaml
      nginx:
        image: nginx:1.25-alpine
        ports:
          - "80:80"
        volumes:
          - ./nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
        depends_on:
          - frontend
          - backend
        restart: unless-stopped
    ```

    IMPORTANT: The nginx.conf is mounted to `/etc/nginx/conf.d/default.conf` — NOT to `/etc/nginx/nginx.conf`. The alpine nginx image includes an http{} block in the base config that already includes `conf.d/*.conf`. The map directive MUST be inside the http block; placing it in `conf.d/default.conf` which is included inside http{} is correct for the server block but the `map` directive needs to be at http context level. To resolve: place the map directive in a separate file, OR use a different approach: define a named location variable via `set` inside the location block instead of top-level map.

    SAFER APPROACH for conf.d/default.conf (avoids map context issue): Use a `set` variable approach:
    ```nginx
    server {
        listen 80;

        location ~* ^/promises/(.+)$ {
            set $backend_proxy 0;
            if ($http_user_agent ~* "(Facebookbot|Twitterbot|TelegramBot|LinkedInBot|Googlebot)") {
                set $backend_proxy 1;
            }
            if ($backend_proxy = 1) {
                proxy_pass http://backend:8000/api/og/promises/$1;
                break;
            }
            proxy_pass http://frontend:5173;
        }

        location / {
            proxy_pass http://frontend:5173;
        }
    }
    ```

    This avoids the map-outside-http context issue when placing in conf.d/default.conf. The `set` + double-`if` pattern works correctly for simple proxy_pass routing.

    Alternatively, if the nginx service uses a top-level nginx.conf mount (at /etc/nginx/nginx.conf), the map directive can be used. Choose the safest approach: mount to /etc/nginx/conf.d/default.conf and use the `set` variable + double-if pattern as documented above.
