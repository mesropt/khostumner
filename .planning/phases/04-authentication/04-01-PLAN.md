<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → FastAPI | All auth inputs (email, password, tokens) cross here — untrusted |
| FastAPI → PostgreSQL | ORM-managed; parameterized queries via SQLAlchemy |
| FastAPI → Mailhog (dev) | Email delivery; dev-only, no production credentials |
| OAuth provider → FastAPI callback | Authorization code + state parameter cross this boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | POST /api/auth/login | mitigate | fastapi-users bcrypt password comparison; no timing oracle |
| T-04-02 | Tampering | CSRF on POST /api/auth/* | mitigate | starlette-csrf double-submit cookie (per D-05); x-csrftoken header required on all state-changing requests |
| T-04-03 | Repudiation | Login / logout events | accept | No audit log in Phase 4; vote history audit added in Phase 7 |
| T-04-04 | Information Disclosure | POST /api/auth/forgot-password email enumeration | mitigate | FastAPI-Users always returns 202 regardless of whether email exists — built-in behavior |
| T-04-05 | Information Disclosure | JWT in httpOnly cookie | mitigate | httpOnly flag prevents JS access (per D-03); Secure flag in production |
| T-04-06 | Denial of Service | Brute-force on /api/auth/login | accept for MVP | Nginx rate limiting already in place; fastapi-users has no built-in rate limit; accept for v1, add account lockout in v2 |
| T-04-07 | Elevation of Privilege | Email verification token replay | mitigate | FastAPI-Users verification tokens are single-use (invalidated on first successful verify call) |
| T-04-08 | Spoofing | OAuth state parameter forgery | mitigate | httpx-oauth generates and validates state parameter automatically |
| T-04-09 | Elevation of Privilege | Token not revoked after logout | accept for MVP | JWTs are stateless; logout clears client cookie only; server-side revocation requires Redis (deferred) |
| T-04-10 | Tampering | Package supply chain (npm/pip installs) | mitigate | All 6 packages verified on PyPI with slopcheck [OK] — see RESEARCH.md Package Legitimacy Audit |
| T-04-SC | Tampering | pip install of auth packages | mitigate | slopcheck [OK] for all packages; no [ASSUMED]/[SUS] packages; no blocking checkpoint required |
</threat_model>
